# インシデント対応計画（AmaSuite / Amazon SP-API）

最終レビュー日: 2026-07-09
次回レビュー予定: 2027-01-09（6ヶ月ごとに見直す）

## 1. 目的と適用範囲

本計画は、AmaSuite（Amazon Selling Partner API を利用する自社利用アプリケーション）が取り扱う
Amazon情報（出品・価格・在庫・注文データ）に関するセキュリティインシデントへの対応手順を定める。

適用範囲:
- SP-API経由で取得したAmazon情報（Cloud Firestore上の保存データ、Cloud Run上の処理）
- SP-APIの認証情報（LWAクライアントシークレット、リフレッシュトークン）
- 上記にアクセスできる開発者端末

## 3. インシデントの定義

以下のいずれかに該当する事象をセキュリティインシデントとして扱う。

- Amazon情報への不正アクセス、漏えい、改ざん、破壊（またはその疑い）
- SP-API認証情報（クライアントシークレット・トークン類）の漏えい（またはその疑い）
- アプリケーション/インフラ（Firebase / Cloud Run / GitHub）への不正侵入の兆候
- 開発者端末の紛失・盗難・マルウェア感染

## 4. 対応手順

1. **検知・初動（検知後ただちに）**
   - 影響範囲の特定（何のデータが・いつから・どこまで影響を受けたか）
   - 被害拡大の防止: 該当認証情報の無効化・ローテーション（LWAシークレット再発行、
     Firebase APIキー制限、Stripe/Keepaキーのローテーション）、必要に応じてCloud Runサービスの停止
2. **Amazonへの報告（検知から24時間以内）**
   - Amazon情報に関連するインシデントは、検知後24時間以内に **security@amazon.com** へ報告する
   - 報告内容: 事象の概要、検知日時、影響を受けたデータの種類と範囲、実施済みの初動対応、連絡先
3. **復旧**
   - 侵入経路の特定と遮断、脆弱性の修正、クリーンな状態からの再デプロイ
   - 全認証情報のローテーション完了確認
4. **事後レビュー（復旧後1週間以内）**
   - 原因分析と再発防止策を本ドキュメントに追記し、必要に応じて手順を更新する

## 5. 連絡体制（24時間体制）

- 個人運営のため、連絡はすべて開発者本人の携帯電話・メールで24時間受けられる体制とする
- Firebase/GCPの障害・セキュリティ通知はアカウント登録メールで受信する
- Amazonからの連絡はセラーセントラル登録メールで受信し、休日を含め毎日確認する

## 6. 予防的対策（平常時）

- 通信は全てTLS。保存データはGoogle Cloudの標準暗号化
- 認証情報は`.env`/環境変数で管理しソースコードにコミットしない
- Firestoreへのアクセスはユーザー本人のみ（Firebase Authentication + user_id認可チェック）
- 開発者端末: OS自動更新・ディスク暗号化（FileVault）・ファイアウォール有効
- 依存パッケージの脆弱性を定期確認（npm audit）
